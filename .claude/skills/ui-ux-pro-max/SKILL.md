# UI/UX Pro Max Skill

Use this skill whenever designing, reviewing, or implementing UI for DoubleA Solar Solutions.

## Design Principles
- Premium Swiss clean energy aesthetic: precise, calm, trustworthy, modern.
- Apple-level restraint: generous whitespace, strong hierarchy, elegant microinteractions.
- Conversion-first: every section must either build trust, explain value, reduce friction, or drive contact.
- Avoid generic SaaS design. Make it feel like a high-end Swiss solar company.
- Do not use fake testimonials or placeholder copy.
- Prefer real trust signals: process clarity, Swiss location, transparent consultation, funding guidance, maintenance, monitoring, professional installation.

## Visual Language
- Background: soft off-white, graphite, deep solar navy, subtle emerald/solar gradients.
- Accents: premium yellow/gold only as controlled energy highlights, not cheap neon.
- Typography: large confident headlines, short precise copy, high readability.
- Cards: glass-like or soft-surface cards with fine borders, shadow discipline, no clutter.
- Motion: framer-motion only for subtle reveal, count-up, hover lift, step transitions. Never overanimate.

## UX Rules
- Primary CTA must always be clear: "Kostenloses Solarangebot erhalten" or "Solarpotenzial berechnen".
- Mobile flow must be better than desktop: no hidden important content, no tiny controls.
- Solar calculator must feel like a premium diagnostic tool, not a generic form.
- Every form must have validation, loading state, success state, error state, and GDPR/privacy notice.
- Use Swiss formatting: CHF, apostrophe thousand separators, German/Swiss wording, no ß; use ss where appropriate.

## Quality Gate
Before finishing:
1. Run lint and typecheck.
2. Check responsive layouts.
3. Check keyboard navigation for forms.
4. Check Lighthouse-minded performance.
5. Check all copy for placeholders.
6. Check all CTAs link to working forms or sections.
7. Check that Supabase env vars are documented.
8. Check that Vercel deploy works.
